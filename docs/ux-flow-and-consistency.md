# UX 흐름 및 UI 일관성 설계 근거

## 1. 사용자 흐름 구성

All4Health는 건강 기록, AI 예측, 조언, 리포트, 챌린지, 펫 보상을 하나의 건강관리 흐름으로 연결하도록 구성했습니다.

핵심 사용자 흐름:

```text
회원가입/로그인
→ 건강설문 또는 건강 수치 입력
→ AI 질환 예측
→ 예측 결과 확인
→ 오늘의 조언/주간 리포트 확인
→ 챌린지 및 펫 보상으로 건강 행동 지속
```

첫 사용자는 건강 상태를 파악하기 위해 회원가입, 이메일 인증, 건강설문 또는 건강 수치 입력을 거치며, 이후에는 홈 대시보드와 사이드바를 통해 주요 기능으로 바로 이동할 수 있습니다.

## 2. 주요 기능 접근 단계

주요 기능은 대부분 3~5회 이내 액션으로 접근할 수 있도록 구성했습니다.

| 기능 | 접근 경로 | 예상 액션 수 |
| --- | --- | ---: |
| 홈 대시보드 | 로그인 후 기본 진입 | 1 |
| 건강 기록 입력 | 홈 → 빠른 기록 → 건강 기록 입력 | 2~3 |
| 식단 기록 입력 | 홈 → 빠른 기록 → 식단 기록 | 2~3 |
| 질환 예측 요청 | 홈 → 질환 예측 → 예측 요청 | 2~3 |
| 예측 결과 확인 | 홈 또는 사이드바 → 질환 예측 → 결과/이력 | 3~4 |
| 오늘의 건강 조언 | 홈 또는 사이드바 → 오늘의 조언 | 2~3 |
| 주간 리포트 | 사이드바 → 리포트 | 2 |
| 챌린지 참여 | 사이드바 → 챌린지 → 상세 → 참여 | 3~4 |
| 내 챌린지 현황 | 사이드바 → 챌린지 → 내 챌린지 | 2~3 |
| 펫 확인 | 사이드바 또는 홈 요약 → 마이펫 | 2~3 |
| 알림 확인 | 상단 알림 또는 사이드바 → 알림 목록 | 1~2 |
| 마이페이지 | 사이드바 → 마이페이지 | 2 |

건강설문, 이메일 인증, 건강 수치 입력은 서비스 특성상 필요한 정보 입력 단계입니다. 불필요한 입력을 늘리기보다는 AI 예측과 리포트 품질을 위한 필수 입력으로 분리했습니다.

## 3. 홈 대시보드 중심의 진입 구조

홈 대시보드는 사용자가 가장 자주 확인하는 기능을 모아 진입 허브 역할을 합니다.

홈에서 확인하거나 이동할 수 있는 항목:

- 오늘의 건강 점수
- 나의 건강 대시보드
- 빠른 건강 기록
- 오늘의 조언
- 질환 예측 요약
- 챌린지 요약
- 뱃지 요약
- 펫 요약

이를 통해 사용자는 사이드바 전체 메뉴를 탐색하지 않아도 주요 기능으로 빠르게 접근할 수 있습니다.

## 4. UI 일관성

전체 서비스는 React SPA 구조로 구성되어 있으며, 공통 레이아웃과 공통 스타일을 기반으로 화면 간 사용 경험을 맞췄습니다.

주요 근거 파일:

- `frontend/src/layouts/AppLayout.tsx`
- `frontend/src/layouts/PublicLayout.tsx`
- `frontend/src/styles/globals.css`
- `frontend/src/api/client.ts`

일관성 기준:

| 항목 | 적용 방식 |
| --- | --- |
| 레이아웃 | 로그인 후 화면은 `AppLayout` 기반 사이드바/콘텐츠 구조 사용 |
| 공개 페이지 | 서비스 소개, 로그인, 회원가입 등은 `PublicLayout` 사용 |
| 메뉴 이동 | 주요 기능은 사이드바와 홈 대시보드 진입점으로 접근 |
| 카드 UI | 대시보드, 기록, 조언, 리포트, 챌린지에서 카드형 패턴 반복 |
| 버튼 | 주요 액션은 강조 버튼, 보조 액션은 서브 버튼으로 구분 |
| 모달 | 경고, 확인, 권한 안내, 챌린지 취소 등은 모달 패턴 사용 |
| 입력 화면 | 건강 기록, 식단, 마이페이지 수정에서 라벨/입력/저장 버튼 구조 반복 |
| API 오류 처리 | 공통 API 클라이언트와 화면별 사용자 안내 문구 사용 |

## 5. 화면별 역할 분리

화면은 사용 목적에 따라 역할을 분리했습니다.

| 영역 | 역할 |
| --- | --- |
| 인증/온보딩 | 사용자 식별, 이메일 인증, 초기 건강 정보 입력 |
| 홈 | 건강 상태 요약과 주요 기능 진입 |
| 건강 기록 | 혈압/혈당, 지질, 신장, 운동, 활동 기록 입력 및 조회 |
| 질환 예측 | 입력값 확인, 예측 요청, 결과 상세, 피드백 |
| 오늘의 조언 | 최근 건강 데이터 기반 조언 확인 및 피드백 |
| 식단 | 식단 입력, 영양성분표 OCR, 식단 기록 관리 |
| 챌린지 | 챌린지 탐색, 참여, 미션 수행, 리더보드 |
| 리포트 | 주간 건강 리포트 조회 및 PDF 내보내기 |
| 마이페이지 | 내 정보, 알림 설정, 약관 동의, 탈퇴 |

## 6. 접근성 및 사용성 고려

구현상 반영한 사용성 요소:

- 건강 기록 입력 화면에서 탭 기반으로 복잡한 입력 항목 분리
- 여러 건강 기록 탭을 한 번에 저장할 수 있는 통합 저장 흐름 제공
- 건강검진표 OCR로 수치 입력 부담 감소
- 영양성분표 OCR로 식단 입력 부담 감소
- 예측 요청 전 입력 부족 상태 안내
- 조언 요청 횟수 제한 시 명확한 안내 문구 제공
- 401 권한 오류 시 회원가입/로그인 필요 안내
- 알림은 목록뿐 아니라 이벤트 발생 시 상단 모달로 즉시 안내
- 마이페이지의 정보 수정, 알림 설정, 약관 동의 관리 접근 흐름 단순화

## 7. 평가 기준 대응 요약

| 평가 항목 | 현재 근거 | 판단 |
| --- | --- | --- |
| 사용자 편의성을 고려한 화면 흐름 | 홈 대시보드, 빠른 기록, 사이드바, 온보딩 흐름 | 주요 기능 3~5단계 이내 접근 가능 |
| 일관된 서비스 경험 | 공통 레이아웃, 공통 CSS, 카드/버튼/모달 패턴 반복 | 디자인·레이아웃·인터랙션 전반적 일관성 확보 |
| 3~5회 이내 메뉴 접근 | 주요 기능별 접근 경로가 사이드바/홈에 집중됨 | 주요 기능 접근이 간편하며 불필요한 입력 최소화 |

## 8. 한계 및 보완 가능점

- 건강 기록 입력, 질환 예측, OCR 기능은 입력 항목 자체가 많아 초보 사용자에게 복잡하게 느껴질 수 있습니다.
- PDF 리포트와 일부 공개 페이지는 일반 앱 화면과 시각적 목적이 달라 스타일 차이가 일부 존재합니다.
- 향후 사용자 테스트를 통해 클릭 수, 이탈 구간, 입력 오류 빈도를 측정하면 UX 개선 근거를 더 강화할 수 있습니다.
